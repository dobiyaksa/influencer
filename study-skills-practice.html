사전 준비 — 환경 세팅

본인의 운영체제에 맞는 탭을 선택하여 환경을 세팅하세요.

🪟 Windows

📦 Mac 필수 설치

Step 1 — 터미널 열기

Cmd + Space로 Spotlight를 열고 터미널을 검색하여 실행합니다.
또는 Finder → 응용 프로그램 → 유틸리티 → 터미널을 실행합니다.

Step 2 — Xcode Command Line Tools 설치

Git이 포함된 개발 도구를 설치합니다. 터미널에 아래 명령어를 입력하세요.

xcode-select --install

팝업이 뜨면 "설치"를 클릭합니다. 이미 설치되어 있으면 무시해도 됩니다.

Step 3 — Node.js 설치

nodejs.org에서 LTS 버전을 다운로드하여 설치합니다.

# 설치 확인
node --version
npm --version
Homebrew 사용자: brew install node로도 설치 가능합니다.
Step 4 — Claude Code 설치

터미널에서 Claude Code CLI를 설치합니다.

npm install -g @anthropic-ai/claude-code
참고: Claude Code를 사용하려면 Anthropic 계정 또는 API 키가 필요합니다.
Step 5 — 프로젝트 다운로드

GitHub에서 프로젝트를 클론합니다.

git clone https://github.com/kimkh10/influencer.git
cd influencer
Step 6 — 로컬 서버 실행

프로젝트를 브라우저에서 확인합니다.

npx serve .

브라우저에서 http://localhost:3000 접속하여 확인합니다.

Step 7 — Claude Code 실행

새 터미널 탭(Cmd + T)을 열고, 프로젝트 폴더에서 Claude Code를 시작합니다.

cd influencer
claude

Claude Code가 실행되면 슬래시 명령어를 사용할 수 있습니다.

스킬 개요 — 사용 가능한 명령어

이 프로젝트에서 사용할 수 있는 슬래시 명령어 목록입니다.

/공구캘린더 인스타그램 공동구매 캘린더 생성 및 수정
/당첨자폼 이벤트 당첨자 배송지 입력 폼 생성 및 수정
/커리큘럼 AI 스터디 커리큘럼 페이지 생성 및 수정
/공구대시보드 공구 매출 대시보드 수정 (API 연동, 차트, 테이블)

사용법: Claude Code 실행 후, 채팅창에 슬래시(/)를 입력하면 사용 가능한 명령어 목록이 나타납니다. 명령어를 선택한 뒤 자연어로 원하는 작업을 설명하면 됩니다.

실습 1 — /공구캘린더

공동구매 캘린더의 디자인과 제품 매핑을 수정해 봅니다.

🎯 실습 목표

  • 캘린더의 테마 색상을 변경할 수 있다
  • 새로운 제품 매핑을 추가할 수 있다
  • 캘린더를 PNG 이미지로 다운로드할 수 있다

🔨 실습 과정

실습 1-A — 테마 색상 변경

Claude Code에서 아래와 같이 입력합니다:

/공구캘린더
# Claude가 응답하면 다음과 같이 요청:
캘린더 테마 색상을 파란색 계열로 변경해줘

브라우저에서 http://localhost:3000을 새로고침하여 변경 사항을 확인합니다.

실습 1-B — 제품 추가

새로운 제품 매핑을 추가해 봅니다:

/add-product
# Claude가 질문하면 아래 정보를 제공:
- 타이틀 키워드: 콜라겐
- 표시 이름: 피쉬 콜라겐
- 설명: 먹는 피부관리 / 저분자 콜라겐
- 배지: NEW
실습 1-C — 자유 수정

본인만의 캘린더를 만들어 봅니다:

/공구캘린더
캘린더 헤더에 있는 핸들 이름을 내 인스타 아이디로 바꿔줘
확인 방법
http://localhost:3000 → 캘린더 확인 → "이미지 저장" 버튼 클릭

실습 2 — /공구대시보드

친한스토어 어드민 API를 연동하여 공동구매 매출 대시보드를 수정합니다.

🎯 실습 목표

  • REST API 로그인(ID/PW + 2FA) 흐름을 이해한다
  • Swagger로 API 스키마를 확인하고 데이터를 연동할 수 있다
  • CORS 프록시를 통해 외부 API를 브라우저에서 호출할 수 있다
  • Chart.js 차트와 테이블을 커스터마이징할 수 있다

📖 핵심 개념

API 인증 흐름

1단계: ID/PW 로그인 → POST /api/admin/v2/authentications/login
2단계: master 계정이면 2FA 발송 → POST /api/admin/v2/authentications/2fa/send
3단계: 슬랙 인증번호 입력 → PUT /api/admin/v2/authentications/2fa/verify
4단계: JWT 토큰 획득 → 이후 모든 API 호출에 Authorization: Bearer {token} 사용

CORS 프록시

브라우저에서 다른 도메인 API를 직접 호출하면 CORS 정책으로 차단됩니다.
Cloudflare Pages Functions가 같은 도메인에서 요청을 받아 서버 측에서 API로 전달합니다.

# 브라우저 → 같은 도메인 (CORS 없음)
influencer-p68.pages.dev/api/admin/...

# Cloudflare Function → API 서버 (서버간 통신, CORS 없음)
→ api.store.friendly-pharmacist.com/api/admin/...
공구 상태 코드

Swagger 스키마 기준 공구 진행 상태:

# 진행예정
T0: 검토중 / T2: 준비중 / T4: 확정 / T6: 진행중 / T8: 진행완료

# 진행중 (주문 발생, 매출 데이터 있음)
Y0: 정산대기

# 완료
Y8: 정산완료

# 기타
C0: 취소 / C4: 보류

🔨 실습 과정

실습 3-B — 폼 필드 추가

새로운 입력 필드를 추가해 봅니다:

관리자 페이지에서 데이터를 확인합니다:

확인 방법
http://localhost:3000/event → 폼 작성 → 제출 → event-admin에서 확인
참고: 실제로 데이터가 저장되려면 Google Apps Script 연동이 필요합니다. 로컬 테스트에서는 UI 변경 확인에 집중하세요.

실습 5 — /deploy (배포하기)

수정한 결과물을 Cloudflare Pages에 배포합니다.

🎯 실습 목표

  • 로컬에서 수정한 파일을 인터넷에 배포할 수 있다
  • 배포된 URL로 모바일에서 확인할 수 있다

🔨 실습 과정

Step 1 — Wrangler 설치

Cloudflare 배포 도구를 설치합니다:

npm install -g wrangler
Step 2 — 배포 실행

Claude Code에서 배포 명령어를 실행합니다:

/deploy
# Claude가 Cloudflare API 토큰을 요청합니다
# 호스트에게 토큰을 받아 입력하세요
Step 3 — 배포 확인

배포 완료 후 제공되는 URL에 접속하여 확인합니다.

주의: Cloudflare API 토큰은 호스트가 제공합니다. 토큰은 절대 외부에 공유하지 마세요. 실습 시 호스트의 안내를 따라주세요.
