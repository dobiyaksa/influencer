인스타그램 공동구매 캘린더(`index.html`)를 생성하거나 수정합니다.

## 주요 기능
- 친한스토어 REST API로 공구 데이터 실시간 연동 (대시보드와 인증 공유)
- Bearer JWT 인증 (ID/PW + Slack 2FA), `dashboard_bearer_token` localStorage 키 공유
- 상태 코드 필터: T4(확정), T6(진행중), T8(진행완료), Y0(정산대기)만 표시
- 인스타그램 4:5 비율(1080x1350) 캘린더 자동 생성
- 제품명/설명/배지 인라인 편집
- 테마 색상/폰트 변경
- html2canvas로 PNG 이미지 다운로드

## 작업 시 참고사항

1. **파일**: `index.html` 단일 파일 (CSS + JS 포함, 파일 분리 금지)
2. **기술 스택**: 순수 HTML + CSS + JavaScript (프레임워크 없음)
3. **CSS 변수**: 테마 색상은 `--accent` (RGB), `--accent-hex` (HEX) 사용
4. **pseudo-element 금지**: html2canvas 호환성 문제로 실제 DOM 요소 사용
5. **토큰 하드코딩 금지**: 사용자가 UI에서 로그인. localStorage에 저장

## API 데이터 매핑
| API 필드 | 캘린더 product 객체 |
|----------|-------------------|
| `group_purchase_plan_idx` | `id` |
| `group_purchase_title` | `title` |
| `start_date` | `startDate` (+ `startDay`, `startYear`, `startMonth` 파싱) |
| `end_date` | `endDate` |
| `image_url` | `thumbnail` |
| (없음) | `salesLink` (빈 문자열) |

## 인증 흐름
1. localStorage에서 `dashboard_bearer_token` 확인
2. 토큰 있으면 자동 로드, 없으면 로그인 가이드 표시
3. 로그인: ID/PW → 2FA(슬랙) → JWT 토큰 발급
4. 대시보드에서 먼저 로그인했으면 캘린더에서 자동 인증

- **새 제품 추가**: `getShortName()`, `getProductDesc()`, `getBadge()` 매핑 수정
- **디자인 변경**: CSS 변수 및 스타일 수정
- **기능 추가/수정**: JavaScript 로직 수정
