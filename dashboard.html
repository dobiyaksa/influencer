🎯 직접 구현해보기 — 약사 전문성 입히기

현재 테이블은 파마브로스 공동구매 API 가 주는 기본 정보만 보여줍니다. 약사 인플루언서로서 제품 효능 · 성분 · 복용 가이드 같은 전문 데이터를 입혀보는 실습 아이디어입니다.

  • 제품별 약사 메모 — 각 행에 효능 · 성분 · 주의사항 · 복용법을 직접 입력하고 localStorage 에 저장. 페이지를 새로고침해도 유지
  • 성분 자동 추출 + 분류 — 제품명에서 성분 키워드(오메가3, 프로바이오, 비타민D, 마그네슘 등) 추출 → 성분 필터/그룹
  • 건강 카테고리 매핑 — 면역 / 다이어트 / 수면 / 관절 / 장 건강 등 카테고리로 분류해 카드 색이나 배지로 시각화
  • 약사 추천 등급 ⭐ — 각 제품에 1~5점 추천 등급 + 코멘트. 평균 점수 높은 제품 TOP 5 panel
  • 제품 상세 모달 확장 — 행 클릭 시 열리는 모달의 openProductDetail 에 효능 / 성분 / 복용법 / 약사 코멘트 섹션 추가
  • 복용 조합 안내 — 함께 먹어도 좋은 / 피해야 할 조합 메모 (약사 전문성 활용). 예: 오메가3 + 비타민E 동시 복용 안내

💡 Claude Code 에 "제품 관리 테이블에 약사 메모 컬럼 추가하고 localStorage 에 저장해줘" 같이 요청하면 빠르게 구현. 데이터가 늘면 별도 functions/api/product-notes 프록시로 옮겨 팀과 공유 가능.

🎯 직접 만들어보기 — 구글 시트를 그대로 대시보드에서 보기

이벤트 응답 데이터는 구글 시트(Google Sheets) 에 자동으로 쌓입니다. 응답자가 이벤트 폼을 작성하면 → Google Apps Script 가 즉시 시트에 한 줄을 추가해요. 지금까지는 그 시트를 열거나 별도 관리자 페이지로 봤다면, 이제 그 시트를 대시보드 안으로 그대로 끌어와 표·카드·검색 같은 걸로 편하게 보여주는 게 이번 실습입니다.

  • 구글 시트와 연결 — 응답 시트에 새 줄이 추가되면 대시보드도 자동으로 따라 보이게
  • 이벤트별 카드 만들기 — 시트 데이터를 이벤트 단위로 묶어 카드로 표시 (이벤트명 · 응답자 수 · 당첨자 수)
  • 당첨자 명단 한눈에 보기 — 카드 클릭 시 시트의 그 이벤트 행만 모아 이름·연락처·배송지 표시
  • 중복 응답 자동 찾기 — 같은 사람이 여러 번 신청한 줄을 시트에서 자동으로 골라 표시
  • 내 양식으로 CSV 내보내기 — 시트의 데이터를 자주 쓰는 택배 신청 양식·배송 라벨 컬럼으로 바로 다운로드

💡 왜 구글 시트? 익숙하니까. 코딩 없이 시트에서 열어보고, 직접 셀을 고치고, 색깔 입히고, 필터도 쓸 수 있어요. 대시보드는 그 시트를 "예쁘게 보여주는 창" 역할만 합니다. 데이터의 진짜 주인은 항상 시트입니다.

Claude Code 에 이렇게 요청해보세요 (그대로 복붙해도 됩니다):

대시보드의 "이벤트 당첨자 관리" 메뉴 안에 구글 시트(이벤트 응답 저장용)의 내용을 카드 + 표로 보여주는 화면을 만들어줘. 데이터는 지금 이벤트 관리자 페이지(event-admin.html)가 쓰는 같은 구글 시트에서 Google Apps Script 를 통해 가져오면 돼.
막막하면 Shift+Tab 으로 plan 모드 진입 후 위 요청을 그대로 던지면 Claude 가 단계별 계획부터 차근차근 보여줍니다.

🎯 직접 구현해보기 — 실습 아이디어

아래 데이터로 다음 기능을 한 번 적용해보세요. cachedPosts 또는 instagramState.posts 배열을 활용하면 됩니다.

  • 프로필 · 팔로워 정보 가져오기 — Apify Actor apify/instagram-profile-scraper 로 본인 프로필 전체를 조회.
    ▸ 숫자 지표: followersCount · followsCount · postsCount
    ▸ 이미지: profilePicUrl · profilePicUrlHD (프로필 사진)
    ▸ 텍스트: fullName · biography · externalUrl (자기소개 + 링크)
    ▸ 배지/속성: verified · private · isBusinessAccount · businessCategoryName
    ▸ 호출 body: { usernames: ['아이디'] }
    ▸ 활용 아이디어: dashboard 상단에 프로필 카드 (사진 + 이름 + 자기소개 + 팔로워 수) / 사이드바 brand 영역에 프로필 사진 / 인증 배지 표시 등
  • 제품 키워드 · 해시태그 필터링 — 캡션의 제품명(예: "오메가3", "프로바이오") 이나 #태그 로 포스트 필터링
  • 자주 쓰는 해시태그 모아보기 — 캡션에서 #태그 추출 + 빈도순 TOP 10 (워드 클라우드 / 막대 차트)
  • 시간대 · 요일 분석 — timestamp 로 어느 요일·시간대에 올린 포스트의 반응이 좋은지
  • 공구 매출 기여도 측정 — 일정 관리 의 공구 plan(title·brand·totalSales) 과 인스타 캡션 키워드를 매칭, 컨텐츠가 어느 공구 매출에 기여했는지 추정
